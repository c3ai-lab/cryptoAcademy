Blockchain und Transaktionen Funktionsweise (Advanced)

Im Folgenden wird die genaue Technologie hinter der Blockchain erklärt, die auf Kryptographischen Berechnungen basiert. Hier wird im speziellen die Technologie erklärt, die bei der Kryptowährung Bitcoin verwendet wird.

Hash-Funktionen sind ein wichtiger Bestandteil der Kryptographie. Hierbei werden aus Datensätzen beliebiger Länge eine Zeichenfolge mit fester Länge generiert. Bei Bitcoin ist der Hash-Algorithmus SHA-256 standard, die Ziffer bezieht sich auf die Länge der Ausgabe. Eine minimale Veränderung des originalen Datensatzes erzeugt einen komplett neuen Hash-Wert. Durch diese Eigenschaften werden Hash-Werte oft als “unique identifiers” verwendet.

Jeder Nutzer hat eine Bitcoin-Adresse (Public Key) und einen privaten Schlüssel (Private Key). Der Public Key ist öffentlich, der Private Key muss geheim gehalten werden. Der Private Key wird verwendet, um eine Transaktion signieren.

Ein Bitcoin-Transaction-Hash wird speziell für eine Transaktion generiert. Dieser Hash wird auch Transaction ID (TXID) genannt und enthält unter anderem:

  • Den Transaktionsstatus
  • Die Höhe des Betrags der Transaktion
  • Bitcoin Adressen des Senders und Empfängers
  • Informationen über die vorherige Transaktion des Coins
  • Zeitpunkt der Transaktion

Nachdem die Transaktion generiert und validiert wurde, wird sie einem Block beigefügt. Jeder Block enthält einen Block Header und Transaktionen. Der Header enthält unter anderem:

  • Version des Blocks
  • Den Hash-Wert des vorherigen Blocks
  • Die Merkle Root
  • Den Zeitpunkt, wann ein Miner den Block versucht zu Minen

Beispiel für einen Block Header:

010000009500c43a25c624520b5100adf82cb9f9 da72fd2447a496bc600b0000000000006cd86237 0395dedf1da2841ccda0fc489e3039de5f1ccdde f0e834991a65600ea6c8cb4db3936a1ae3143991

Der erste Block der Kette, der Genesis Block wurde zu Beginn der Währung fest im Quellcode verankert. Dieser enthält keinen Hash-Wert eines vorherigen Blocks. Um den Hash-Wert des aktuellen Blocks zu ermitteln, wird der Block Header zweimal nach SHA-256 gehasht.
